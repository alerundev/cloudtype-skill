# 프리티어 리소스 (Free Tier)

Cloudtype 에는 두 종류의 리소스가 있고, **PUT 페이로드의 `resources.spot` 으로 선택**합니다.

- `resources.spot: true` → **프리티어 리소스** (무료 풀)
- `resources.spot: false` → **구독 리소스** (Hobby / Pro 풀에서 차감)
- 키 자체 생략 → 서버 디폴트 (계정 상황에 따라 결정)

---

## 부여 조건

- 카드를 등록하면 **1GB 의 프리티어 리소스가 자동 부여**됩니다.
- 카드 미등록 상태에서 배포를 시도하면 `limit is 0` 오류가 발생합니다.
- 구독 플랜(Hobby / Pro) 사용자도 **프리티어 풀(1GB)을 함께 사용할 수 있습니다.**

---

## 제약사항

### 리소스 한도
| 항목 | 한도 |
|---|---|
| 메모리 | **1GB** |
| 임시 디스크 | **1GB** (DB 슬롯 1개) |
| 서비스 | **4개** |
| 프로젝트 | **1개** |
| 팀원 | 최대 4인 |

### 가용성
- **매일 1회 서비스 자동 정지** — 오전 3~9시 사이의 특정할 수 없는 시점.
- **임시 메모리 할당**으로 언제든 재시작 가능. 재시작 중 몇 분간 접속 불가.
- 불특정 시점 다운타임 발생 가능 → **운영용 ❌ 테스트용 ✅**
- 무중단 배포는 지원됩니다.

### 기능 제한
- **커스텀 도메인 연결 불가** (프리뷰 도메인만)
- **레플리카(동시실행) 적용 불가**
- **프라이빗 이미지 저장소 연결 불가**
- 업로드 한도 **100MB** (초과 시 413 오류)

### 네트워크 성능
- 대역폭 **10Mbps**
- 응답 제한시간 **60초**
- 최대 커넥션 **60개**
- 초당 요청 **60개** / 분당 요청 **600개**

## 사용자 의도 → `spot` 매핑

스킬이 PUT 페이로드를 만들 때:

| 사용자 발화 예 | `resources` 처리 |
|---|---|
| "프리티어로", "무료로", "free tier", "테스트용으로", "임시로 띄워봐" | `{ "spot": true }` 명시 |
| "구독 리소스로", "운영용으로", "프로덕션으로", "무중단으로" | `{ "spot": false }` 명시 |
| 명시 없음 | **`resources` 자체를 생략** (서버 디폴트) |

**핵심 원칙**: 사용자 명시가 없는 경우 `spot: false` 같은 값을 디폴트로 임의 삽입하지 않습니다. 호출자 계정에 맞는 서버 디폴트를 따릅니다.

---

## 완료 보고 시 검증

배포 응답의 `resources.spot` 을 사람말로 변환해서 사용자에게 보고합니다.

- `true` → "프리티어 리소스로 배포됨"
- `false` → "구독 리소스로 배포됨"
- 응답에 키가 없으면 `GET /project/.../deployment/<name>` 으로 재조회.

사용자가 "프리티어로" 요청했는데 응답에 `spot: false` 가 돌아온 경우 (또는 그 반대), **완료로 보고하지 않고 즉시 사용자에게 알려** 결정을 기다립니다.

---

## 플랜 전환 시 주의

Free Tier 에서 Hobby / Pro 로 구독 전환을 해도, **이미 프리티어로 배포된 서비스의 리소스는 자동으로 구독 리소스로 전환되지 않습니다.**

전환하려면:
1. 콘솔에서 "프리티어" 라벨이 붙은 서비스의 설정으로 이동
2. 리소스 유형을 구독으로 변경 후 재배포

(이 작업은 사용자가 콘솔에서 직접 수행합니다. 스킬이 자동으로 PUT 을 보내지 않습니다.)
